测试结果分析：

  1. Naive版本（每个线程直接atomic）
    - 1M数据：2.48ms
    - 性能最差，随着数据规模增大急剧下降
  2. Block版本（block级reduce + atomic）
    - 1M数据：0.048ms
    - 相比Naive版本：51倍加速
    - 通过在block内先做reduce，大幅减少了atomic操作次数
  3. Warp版本（warp shuffle + block级reduce + atomic）
    - 1M数据：0.034ms
    - 相比Naive版本：73倍加速
    - 相比Block版本：1.42倍加速
    - 使用warp shuffle进一步优化了reduce过程
